# Ручное тестирование сервиса F‑Bank

Окружение: Windows 10, браузер Chrome/Edge (актуальная версия), приложение открыто локально по `http://localhost:8000/`.

## TC-01 — Успешный перевод валидной суммы (позитивный)

- **Цель**: убедиться, что валидный перевод выполняется.
- **Предусловия**: приложение открыто; выбран счёт (например «Рубли»).
- **Шаги**:
  1. Выбрать счёт «Рубли».
  2. В поле «Номер карты» ввести валидный номер (16 цифр), например `1111 1111 1111 1111`.
  3. В поле «Сумма перевода» ввести `1000`.
  4. Нажать кнопку «Перевести».
- **Ожидаемый результат**: появляется уведомление об успешном принятии перевода банком (сумма и валюта корректные).
- **Фактический результат**: (заполнить по результату прогона).

## TC-02 — Недостаточно средств (негативный, без дефекта)

- **Цель**: убедиться, что при недостатке средств перевод блокируется.
- **Предусловия**: приложение открыто по `http://localhost:8000/?balance=100&reserved=0`; выбран счёт «Рубли».
- **Шаги**:
  1. Открыть `http://localhost:8000/?balance=100&reserved=0`.
  2. Выбрать счёт «Рубли».
  3. Ввести номер карты `1111 1111 1111 1111`.
  4. Ввести сумму перевода `1000`.
- **Ожидаемый результат**:
  - отображается сообщение «Недостаточно средств на счете»;
  - кнопка «Перевести» не отображается/недоступна.
- **Фактический результат**: (заполнить по результату прогона).

## TC-03 — Перевод 0 ₽ (дефект BUG-01)

- **Цель**: проверить валидацию суммы перевода.
- **Предусловия**: приложение открыто; выбран счёт «Рубли»; введён валидный номер карты.
- **Шаги**:
  1. Выбрать счёт «Рубли».
  2. Ввести номер карты `1111 1111 1111 1111`.
  3. Ввести сумму перевода `0`.
  4. Попробовать выполнить перевод.
- **Ожидаемый результат**:
  - перевод запрещён (кнопка неактивна/не отображается), либо показана ошибка «Сумма должна быть больше 0».
- **Фактический результат**:
  - сервис принимает перевод и показывает успешное уведомление «Перевод 0 ₽ … принят банком!» (BUG-01).

## TC-04 — Перевод отрицательной суммы (дефект BUG-02)

- **Цель**: проверить запрет отрицательных сумм.
- **Предусловия**: приложение открыто; выбран счёт «Рубли»; введён валидный номер карты.
- **Шаги**:
  1. Выбрать счёт «Рубли».
  2. Ввести номер карты `1111 1111 1111 1111`.
  3. Ввести сумму `-20000`.
  4. Наблюдать расчёт комиссии и доступность действия «Перевести».
  5. Попробовать выполнить перевод.
- **Ожидаемый результат**:
  - отрицательная сумма запрещена;
  - комиссия не должна становиться отрицательной;
  - перевод не выполняется, отображается сообщение об ошибке.
- **Фактический результат**:
  - сервис принимает перевод и показывает успешное уведомление, комиссия становится отрицательной (BUG-02).

## TC-05 — Перевод USD больше баланса (дефект BUG-03)

- **Цель**: убедиться, что проверка доступного баланса выполняется корректно для выбранной валюты.
- **Предусловия**: приложение открыто; выбран счёт «Доллары»; введён валидный номер карты.
- **Шаги**:
  1. Выбрать счёт «Доллары» (на экране отображается баланс 100 $).
  2. Ввести номер карты `1111 1111 1111 1111`.
  3. Ввести сумму перевода `1000`.
  4. Проверить, появляется ли блокировка по недостатку средств.
- **Ожидаемый результат**:
  - перевод суммы, превышающей баланс, отклонён;
  - отображается «Недостаточно средств на счете»;
  - успешное выполнение операции невозможно.
- **Фактический результат**:
  - сервис позволяет выполнить перевод 1000 $ и показывает успешное уведомление (BUG-03).
